# TaskFlow API — CSDD Example

Este directorio contiene un ejemplo completo de aplicación de la metodología **Contract-first Spec-Driven Development (CSDD)** para una API de gestión de tareas.

## Cómo leer este ejemplo

Sigue el orden de los archivos para entender el flujo de diseño:

1. **[CONSTITUTION.md](CONSTITUTION.md)**: El manifiesto y las reglas del juego.
2. **[contracts.md](contracts.md)**: La definición de tipos, interfaces y reglas de validación.
3. **[spec.md](spec.md)**: Las historias de usuario y criterios de aceptación.
4. **[plan.md](plan.md)**: La arquitectura, dependencias y esquema de base de datos.
5. **[tasks.md](tasks.md)**: El desglose de tareas atómicas para el desarrollo.

## Por qué este ejemplo es importante

Muestra cómo la especificación se convierte en el contrato autoritativo. Si un agente de IA recibiera estos documentos, tendría todo lo necesario para construir el sistema sin "alucinar" o inventar librerías, garantizando que el código final es exactamente lo que el humano diseñó.
